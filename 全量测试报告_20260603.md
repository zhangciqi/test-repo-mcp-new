## 六、缺陷详情

### BUG-001 【低】ACH_003 速度之王成就时间判断存在精度偏差

- **严重程度**: 低
- **缺陷位置**: [js/game.js L47](file:///c:/Users/Administrator/Desktop/0603/test-repo-mcp-new/js/game.js#L44-L48)
- **缺陷类型**: 逻辑问题

**问题代码**:
```javascript
{ id: 'ACH_003', name: '速度之王', desc: '60秒内合成1024', icon: '⚡', 
  check: (value) => value >= 1024 && this.elapsedSeconds <= 60 }
```

**复现步骤**:
1. 开始游戏，快速合成到 1024
2. 在 59.9 秒时完成合并（此时 `elapsedSeconds` 可能仍为 59，因为定时器每秒更新一次）
3. 成就正常触发，但存在 1 秒精度误差

**现象**: `elapsedSeconds` 由 `setInterval` 每秒更新一次，而合并检测在 `executeMove` 中同步执行。两者之间无同步保证，导致判断可能基于过时的秒数（最多偏差 1 秒）。

**预期结果**: 应使用 `this.getElapsedSeconds()` 实时计算精确秒数，确保判断准确。

**修改建议**:
```javascript
// 修改前
check: (value) => value >= 1024 && this.elapsedSeconds <= 60
// 修改后
check: (value) => value >= 1024 && this.getElapsedSeconds() <= 60
```

### BUG-003 【低】键盘事件监听器 passive 模式与 preventDefault 冲突

- **严重程度**: 低
- **缺陷位置**: [js/game.js L251](file:///c:/Users/Administrator/Desktop/0603/test-repo-mcp-new/js/game.js#L251)
- **缺陷类型**: 代码规范问题

**问题代码**:
```javascript
document.addEventListener('keydown', (e) => this.handleKeydown(e), { passive: true });
```

**现象**: `passive: true` 声明的监听器中调用 `e.preventDefault()` 会被浏览器静默忽略，控制台可能输出警告。

**影响范围**: 在支持 passive 事件且默认行为会触发页面滚动的浏览器中，方向键可能同时触发游戏移动和页面滚动。

**预期结果**: 应使用 `{ passive: false }` 以允许 `preventDefault()` 生效。

**修改建议**:
```javascript
// 修改前
document.addEventListener('keydown', (e) => this.handleKeydown(e), { passive: true });
// 修改后
document.addEventListener('keydown', (e) => this.handleKeydown(e), { passive: false });
// 或直接省略 passive（默认 false）
document.addEventListener('keydown', (e) => this.handleKeydown(e));
```
